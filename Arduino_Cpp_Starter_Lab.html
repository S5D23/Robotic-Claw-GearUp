01

Variables & Data Types

Teaching the computer to remember things

variablesdata typesconstcommentsSerial output

What it doesStores six different kinds of value, prints them to the Serial Monitor, then blinks the on-board LED while counting each blink.

Watch forLook at the two division lines. 7 / 2 with whole numbers gives 3 — the leftover is thrown away. The same math with decimals gives 3.50. The type you pick changes the answer.

The C++ idea

A variable is a labelled box that holds one value. The type (int, float, bool, char, String) tells C++ what kind of thing lives in the box and how much room to set aside. A const is a box that is locked shut once it is filled.

Try it yourself

  1. Change batteryVolts to 11.1 and watch the printout update.
  2. Add a new int yearFounded = 2024; and print it.
  3. Make the LED blink faster by lowering the delay(500) number.
02

Functions & Loops

Write it once, use it everywhere

functionsparametersreturn valuesfor-loopwhile-loopscope

What it doesDefines a reusable blink() function and an addUp() function that returns a total, then runs a blinking launch countdown.

Watch forblink(3, 400) and blink(5, 100) call the same function with different inputs. One function, many behaviours.

The C++ idea

A function is a named recipe for a job. Parameters are the ingredients you hand in; a return value is what you get back. A for-loop repeats a known number of times; a while-loop repeats until a condition stops being true. Variables made inside a function (its scope) vanish when the function ends.

Try it yourself

  1. Call blink(10, 60) for a burst of fast blinks.
  2. Change addUp(1, 10) to addUp(1, 100). What is the total?
  3. Start the countdown at 5 instead of 3.
03

Conditions & Operators

Teaching the program to make choices

if / elsecomparison&& || !modulo %Serial input

What it doesReads a number you type into the Serial Monitor and sorts it: positive/negative/zero, even/odd, and blinks if it is an even number from 1 to 100.

Watch forSet the Serial Monitor's line ending to Newline, type a number, press Enter. The % operator gives the remainder — that is how the code knows if a number is even.

The C++ idea

An if statement runs code only when a condition is true; else if and else handle the other cases. Comparison operators (>, <, ==) ask yes/no questions. Logical operators glue questions together: && (and), || (or), ! (not).

Try it yourself

  1. Add an else if (number > 1000) branch that prints "huge!".
  2. Change the range check to 1–50 instead of 1–100.
  3. Make odd numbers blink instead of even ones.
04

Arrays

One name, many values

arraysindexingsizeof lengthiterating a list

What it doesStores a Morse-code pattern in an array and steps through it with a loop, blinking S O S on the on-board LED.

Watch forIndexes start at 0, not 1. An array of 9 items uses indexes 0 through 8. The sizeof trick measures the list's length for you.

The C++ idea

An array is a row of boxes that share one name. You reach each box by its index, counting from 0. A for-loop is the natural partner of an array: it walks the index from the start to the end so you can touch every value in turn.

Try it yourself

  1. Change unit from 200 to 400 to slow the Morse down.
  2. Build a new pattern, like {1, 3, 1, 3}, and watch it play.
  3. Spell a different letter: M is two dashes {3, 3}.
05

Structs

Design your own data type

structgrouping dataarray of structsdot accessbool return

What it doesModels the four joints of a robot arm as a custom Joint type, prints a safety report, then moves the claw out of range to show the check fire.

Watch forThis is the real idea behind your Claw Master arm: every joint has a safe minAngle and maxAngle. Pushing the claw to 200° trips the OUT OF RANGE warning.

The C++ idea

A struct bundles related values into one new type you invent. Here a Joint carries a name plus three angles. You read a field with a dot: arm[3].angle. Put structs in an array and you can describe a whole robot — exactly how real firmware tracks each servo.

Try it yourself

  1. Add a fifth joint, like a {"Wrist", 0, 180, 90}.
  2. Change a starting angle so a different joint reads OUT OF RANGE.
  3. Add a speed field to the struct and print it in the report.

How to teach these: spend one session per lesson. Run the code first so students see it work, then change one number together and re-upload — seeing cause and effect is the whole lesson. The "Try it yourself" tasks are deliberately small so a beginner can succeed on the first attempt.

The lessons build on each other: data → functions → decisions → lists → your own data types. By Lesson 5 students are modelling a real robot arm — the same idea behind GearUp's Claw Master.
